# Treasury Report — Design Spec

## Overview

A printable, formal environmental fee collection report for the City Treasury Office, accessible from the admin panel. Renders in-browser with a "Download PDF" button that triggers the browser print dialog.

## Goal

Provide treasury-relevant financial reporting on environmental fee collections, suitable for government record-keeping and stakeholder demos.

## Architecture

### New API Route: `GET /api/reports/treasury`

Combines fee stats, status breakdown, and recent transactions in one call.

- **Auth:** ADMIN role + `environmentalFees` permission (same pattern as `/api/fees/stats`)
- **Query params:** `from` (ISO date), `to` (ISO date)
- **Response shape:**

```ts
{
  summary: {
    totalRevenue: number;
    totalVisitors: number;
    totalPayments: number;
    collectionRate: number; // active / (active + expired + failed) as percentage
  };
  byPayerType: {
    payerType: string;
    count: number;
    unitRate: number;
    totalCollected: number;
    sharePercent: number;
  }[];
  byStatus: {
    status: string;
    transactions: number;
    amount: number;
    percent: number;
  }[];
  recentPayments: {
    referenceId: string;
    payerName: string;
    totalPersons: number;
    totalAmount: number;
    status: string;
    paidAt: string | null;
  }[];
}
```

### New Page: `/admin/treasury-report`

Lives at `app/(admin)/admin/treasury-report/page.tsx`. Uses the existing admin layout (sidebar visible on screen, hidden in print).

**Screen-only controls (hidden in print):**
- Date range picker (reuses `DateRangeFilter` component)
- "Download PDF" button → `window.print()`

**Report sections (visible in both screen and print):**

1. **Header** — "City of Puerto Princesa" / "Environmental Fee Collection Report" / date range / "Generated: [timestamp]"
2. **Executive Summary** — Three key metrics: Total Revenue Collected, Total Visitors Served, Collection Rate %
3. **Revenue by Payer Type** — Table: Type | Count | Unit Rate (PHP) | Total Collected | % Share
4. **Collection Status** — Table: Status | Transactions | Amount | % of Total
5. **Fee Schedule Reference** — The 5 payer types with rates from `fee-constants.ts`
6. **Recent Transactions** — Table: Ref ID | Payer | Persons | Amount | Status | Date (last 50)
7. **Footer** — "Per City Ordinance No. 1058" / "Generated by ZircuVia Environmental Management System"

**Print CSS:** `@media print` styles via Tailwind's `print:` variant:
- Hide sidebar, date picker, buttons
- White background, proper margins
- Clean table borders

## Dependencies

- Reuses: `DateRangeFilter`, `formatCurrency`, `formatDate`, `parseDateRangeFilter`, `PAYER_TYPE_LABELS`, `FEE_PRICES`
- No new packages needed
- `window.print()` for PDF generation (no PDF library)

## Out of Scope

- Real PDF generation library (jsPDF, react-pdf)
- Email/scheduled report delivery
- Historical trend charts
- Digital signatures
